# 스내피즘 매출 대시보드 자동화

> 스내피즘(Snapism) 매출 데이터 자동 수집 + Streamlit 실시간 대시보드

---

## 🎯 목적

어드민 CMS에서 일별 결제 데이터를 자동 크롤링하여 누적 저장하고,
KPI 목표 달성률·IP 정산·포토이즘 매출을 한 화면에서 모니터링한다.

---

## ✅ 효과

- 매일 09:00 자동 수집 → 수동 CSV 다운로드 불필요
- 국내/해외·팀별 KPI 달성률 실시간 확인
- IP별 Jira RS율 기반 정산액 자동 계산
- SMBS 매매기준율 기반 다통화 KRW 환산

---

## 📊 데이터 검증

### 📥 Raw 데이터 출처

| 소스 | 수집 방식 | 저장 위치 | 주기 |
|---|---|---|---|
| **스내피즘 어드민 CMS** | Playwright 크롤러 (`crawler.py`) | `data/master.csv` | 매일 09:00 자동 |
| **Jira CANDIP 프로젝트** | REST API `/rest/api/3/search/jql` (`jira_client.py`) | `data/jira_rs_cache.json` | 1시간 캐시 |
| **환율 — 오늘** | SMBS 스크래핑 (`update_rates.py`) | `config.json > exchange_rates` | 매일 09:00 자동 |
| **환율 — 과거날짜** | fawazahmed0 날짜별 API (`update_rates.py`) | `data/rates_date_cache.json` | 조회 시 캐시 |
| **IPX 엑셀 KPI** | 수동 업로드 — `2026 IPX MASTER DATA.xlsx` REPORT 시트 | `data/kpi_targets.csv` / `data/kpi_actuals.csv` | 월별 수동 |

> 📸 포토이즘 어드민 데이터는 별도 문서 참조: [`포토이즘-매출대시보드.md`](포토이즘-매출대시보드.md)

**Jira JQL (RS율 조회)**
```jql
project = CANDIP
AND issuetype = Sub-task
AND summary ~ "프로그램 및 검수"
AND "브랜드[select list (multiple choices)]" IN (Snapism, "사용 X (구 'Sticker')")
AND status IN ("할 일", "진행 중", "송출 중", "완료", "TEST 맵핑", "검수 완료", "배포 완료", "In Review", "리소스 업로드 완료")
ORDER BY created DESC
```

**사용 Jira 필드**
- `customfield_[rs_agency_field]` — 소속사 RS율
- `customfield_[rs_mgmt_field]` — 대행사 RS율
- `customfield_[wbs_field]` — WBS (IP명 추출)
- `duedate` — 종료일 (환율 기준일 결정)
- `parent.summary` — 부모 태스크 제목 (IP명 fallback)

---

### 🔍 필터링·비교 방법

**CMS 매출 데이터**
- `취소 여부 = False` → 취소 건 전체 제외
- `최종 결제 금액 >= 0` → 음수 건 제외
- 쿠폰 100% 건: `최종 결제 금액 = 0 AND 쿠폰 할인 금액 > 0` → 쿠폰 집계에 별도 포함
- **국내/해외 분리**: `국가코드 == 'kr'` (국내) vs `국가코드 != 'kr'` (해외)
- **다통화 KRW 환산**: `KRW환산 = 최종결제금액 × 환율` (SMBS 매매기준율 기준)

**KPI 엑셀 파싱 (REPORT 시트)**
- 목표 행 탐색: `col[1] == segment AND col[2] == "목표"`
- 실적 행 탐색: 목표 행 이후 5행 이내에서 `col[2] == "실적"`
- 세그먼트: `TTL / 국내 / 해외 / A팀 / C팀 / 오리지널`

**IP 정산 블렌딩 (KPI 목표 페이지)**
- 엑셀 확정값 우선: `kpi_actuals.csv`에서 해당 세그먼트 + 실적 > 0인 달
- 나머지: CMS 실시간 집계로 대체

**환율 기준일 결정 (IP 정산 페이지)**
- Jira `duedate <= 오늘` → 종료일 기준 환율 (fawazahmed0 날짜별 API)
- `duedate > 오늘` 또는 미설정 → 오늘 기준 환율 (SMBS)

### ✅ 검증 가능 예시

**① CMS 건수 검증**
```
어드민 CMS → 날짜 필터 동일 기간 설정 → CSV 다운로드 후 행수 확인
= data/master.csv 해당 날짜 필터 행수 (취소 건 포함)
```

**② KPI 실적값 검증**
```
엑셀 REPORT 시트 → TTL 실적 행의 해당 월 값
= KPI 목표 페이지 → 전체 탭 → 해당 월 실제매출
```
> 엑셀 업로드 후 확정된 달: 엑셀값 그대로 표시  
> 미확정 달(당월 등): CMS 실시간 합산값 표시 → 엑셀값과 소폭 차이 가능

**③ Jira RS율 검증**
```
Jira CANDIP → 위 JQL 실행 → 해당 IP Sub-task → 소속사 RS / 대행사 RS 필드 확인
= IP 정산 페이지 사이드바 "Jira 원본" 표시값
```

**④ 환율 검증**
```
SMBS (http://www.smbs.biz/ExRate/TodayExRate.jsp) → 오늘 날짜 JPY/CNY 매매기준율
= IP 정산 페이지 하단 "환율 기준" 캡션값
```

### ⚠️ 알려진 제외 케이스

| 케이스 | 처리 방식 |
|---|---|
| **취소 건** | `취소 여부 = True` 전체 제외 |
| **쿠폰 100% 건** | 매출 집계 제외, 쿠폰 집계에만 포함 |
| **CNH (역외 위안화)** | CNY와 동일 환율 적용 (SMBS는 CNH만 제공) |
| **엑셀 미업로드 월** | 해당 달은 CMS 실시간 집계로 자동 대체 |
| **팀별 데이터 (A팀/C팀/오리지널)** | CMS 블렌딩 없음 — 엑셀 전용. 엑셀 미업로드 시 데이터 없음 |
| **Jira RS 캐시** | 1시간 재사용. Jira 수정 직후 최대 1시간 지연 가능 |
| **환율 캐시 (과거 날짜)** | 한 번 조회 후 영구 캐시. 소급 수정 시 `data/rates_date_cache.json` 해당 날짜 삭제 필요 |
| **SMBS 접속 실패 시** | fawazahmed0 API로 자동 fallback |
| **주말 종료일** | `get_effective_date()` 로 이전 영업일로 자동 보정 |
| **PEN, LAK** | CURRENCIES 목록에 있으나 SMBS 미지원 → fawazahmed0 기본값 사용 |

## 🛠 운영 메모

- 스케줄러 실행: `pythonw scheduler.py` (백그라운드) 또는 `대시보드실행.bat`
- PC 재시작 후 자동 재개: `대시보드_자동시작.bat` 시작프로그램 등록
- KPI 엑셀 업로드: 대시보드 사이드바 → 파일 업로드 → TTL/국내/해외/A팀/C팀/오리지널 6개 세그먼트 자동 파싱
- 환율 수동 갱신: `python update_rates.py`
- Jira 캐시 강제 갱신: `python jira_client.py` (직접 실행)
