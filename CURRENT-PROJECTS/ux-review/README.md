## 전 페이지 공통 패턴 (우선 처리 권장)

6개 페이지 모두에서 같은 문제가 반복됩니다. 개별 문구보다 **이 4가지를 일괄로**
잡는 게 효율적입니다.

### 1. 톤 혼용 — 해요체로 통일 (원칙 ①)
라벨·본문·일부 캡션은 `~해요`체인데 **빈 상태·오류·일부 피드백만 `~습니다`/명사형**이라
화면 간·화면 내 톤이 깨집니다. 로그인/승인 화면(`auth.py`)이 이미 해요체이므로 그쪽에
맞춰 통일하는 게 좋습니다.
- 예: `데이터가 없습니다.` → `아직 표시할 데이터가 없어요.`

### 2. 빈 상태가 "없음"만 말함 (원칙 ②·⑤ · 빈 상태 규칙)
거의 모든 페이지의 빈 상태가 상태만 알리고 **다음 행동을 안 줍니다**. "왜 비었는지 +
무엇을 하면 되는지"를 긍정형으로 덧붙입니다.
- 예: `상세 데이터 없음` → `아직 볼 상세 내역이 없어요. 기간이나 필터를 넓혀 보세요.`

### 3. 구현명·기술명 노출 (원칙 ③)
사용자에게 내부 구현명이 그대로 보입니다.
- `raw 폴더`·`데이터추가.bat`(스내피즘), `config.json`·`jira`(IP정산), `YoY`·`파싱`(KPI),
  `duedate`(기간후), `Gemini 2.5 Flash + Google Search Grounding`·`(fallback)`(주간리포트)
- 목적 중심 표현으로: `파싱` → `불러오기`, `YoY` → `전년비`, `duedate` → `판매 기한` 등.

### 4. 같은 대상을 두 이름으로 (원칙 ⑦)
- 포토이즘: 사이드바 `IP명 선택` ↔ 본문 `프레임 (IP)` — 한 이름으로.
- 기간후: 사이드바 `최소 기한후 금액` ↔ 본문 `기한 초과` — `기한 초과`로 통일.

## 적용 우선순위 (가이드 「적용 우선순위」 기준)

1. 빈 상태·오류 메시지 — 다음 행동 추가 + 해요체 (전 페이지, 임팩트 최대)
2. 구현명·기술명 노출 제거
3. 페이지 내 용어 통일(포토이즘·기간후)
4. 저장/완료 피드백에 다음 행동 한 줄
5. 버튼 모호어(`처리`·`실행`·`초기화`) 다듬기

각 페이지 파일에 위치(클릭 가능한 `파일:라인`)·현재 문구·수정 제안(복사용 완성형)이
표로 정리돼 있습니다.
